# Qualidade e Testes (arquivo)

> Plano: [Roadmap integrado](../02-planejamento/roadmap-spec.md#marcos-principais)
> Changelog: [/CHANGELOG/20251120103000.md](/CHANGELOG/20251120103000.md)

As diretrizes oficiais de validação residem na fase [04-Testes e Validação](../04-testes-e-validacao/README-spec.md). Este diretório preserva artefatos herdados e atalhos para métricas históricas, garantindo que indicadores de sustentabilidade, disponibilidade e governança sigam rastreáveis. 【F:docs/rup/02-planejamento/requisitos-spec.md†L140-L220】

**Requisitos associados:** [REQ-015](../02-planejamento/requisitos-spec.md#req-015), [REQ-017](../02-planejamento/requisitos-spec.md#req-017), [REQ-022](../02-planejamento/requisitos-spec.md#req-022).
**Integração colaborativa:** utilize estes arquivos para sincronizar indicadores com o painel de SLA descrito em [REQ-034](../02-planejamento/requisitos-spec.md#req-034) e com o fluxo de aprovação compartilhado (`REQ-031` a `REQ-035`).

**Novos requisitos mapeados a partir de anexos:**
- **Gestão de perfis multientidade** — cadastros completos para organizações, unidades, lotes, logística e compliance, com status de verificação e perfis acumuláveis. 【F:docs/rup/99-anexos/Requisitos_Banco_Digital.txt†L6-L185】
- **Marketplace com passaporte digital e escrow** — publicação de lotes com rastreabilidade, filtros avançados, custódia financeira e transferência segura entre contas. 【F:docs/rup/99-anexos/Requisitos_Banco_Digital.txt†L46-L226】
- **Relatórios ESG e fiscais auditáveis** — geração de relatórios com hash verificável, metodologias padrão e extratos fiscais completos para usuários e investidores. 【F:docs/rup/99-anexos/Requisitos_Banco_Digital.txt†L245-L399】
