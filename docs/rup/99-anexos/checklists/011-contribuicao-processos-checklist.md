# Checklist de Contribuição e Colaboração

## Fluxo de Trabalho
- [ ] Confirmar que contribuidores seguem o fluxo de branches definido (feature → PR → revisão → merge em `main`).
- [ ] Garantir que PRs incluem descrição completa, referência a requisitos/riscos e evidências de testes.
- [ ] Verificar que reviews humanos e automatizados são registrados antes do merge.

## Convenções de Commit
- [ ] Checar que mensagens de commit utilizam formato acordado (prefixos, escopo, referências a IDs quando necessário).
- [ ] Validar que commits agrupam alterações relacionadas e evitam mudanças não solicitadas.

## Documentação do Contribuidor
- [ ] Garantir que guias para onboarding de contribuidores estejam atualizados (acesso, configuração local, scripts obrigatórios).
- [ ] Confirmar que checklists de PR, templates de issue e políticas de governança estão disponíveis na pasta `docs/` apropriada.
