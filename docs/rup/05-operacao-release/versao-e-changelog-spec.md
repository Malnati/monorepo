# Versão e Changelog (Histórico)

> Base: [./versao-e-changelog.md](./versao-e-changelog.md)
> Referência atual: [Publicação e Versionamento](../05-entrega-e-implantacao/publicacao-e-versionamento-spec.md)

O controle de versões do App foi centralizado no documento de [Publicação e Versionamento](../05-entrega-e-implantacao/publicacao-e-versionamento-spec.md) e no diretório `CHANGELOG/`. Esta página permanece apenas para manter rastreabilidade com [REQ-019](../02-planejamento/requisitos-spec.md#req-019), [REQ-022](../02-planejamento/requisitos-spec.md#req-022) e [REQ-029](../02-planejamento/requisitos-spec.md#req-029), além de registrar os legados equivalentes (`REQ-019`, `REQ-022`, `REQ-029`).

Ao criar novos registros de versão, certifique-se de:
- apontar requisitos atuais e os legados correspondentes (`REQ-001`…`REQ-035`) no changelog;
- vincular evidências de build em `docs/checklists/build-log.md` e auditorias colaborativas quando houver ativação de features REQ-304;
- atualizar os índices de requisitos e riscos sempre que uma versão alterar compromissos regulatórios.
